\# 🏢 Syndiceo

\*\*Syndiceo\*\* е интегрирана софтуерна система за управление на етажна
собственост, разработена с цел дигитализация и автоматизация на
административните, финансовите и комуникационните процеси в жилищни
сгради.

Проектът заменя остарелите ръчни методи като хартиени тетрадки, отделни
Excel таблици и неструктурирана комуникация, като предоставя
централизирана, сигурна и удобна среда за работа както за
администратори, така и за живущи.

---

\## 📌 Описание на проекта

Syndiceo е изградена като цялостна система за управление на входове,
апартаменти, собственици, задължения, приходи, разходи, ремонти, сигнали
и документи.

Системата позволява по-прозрачно управление на общите части, по-лесно
проследяване на плащанията и по-добра комуникация между домоуправителите
и живущите.

\## 🚀 Основни функционалности

\### 🏠 Управление на собствеността

\- Управление на адреси, блокове и входове - Добавяне и редактиране на
апартаменти - Свързване на апартамент със собственик - Проследяване на
брой живущи в апартамент - Отбелязване на специален статус на
апартамент - Потвърждаване на апартамент от потребител

Йерархичната структура е организирана по следния начин:

\`\`\`text Address → Block → Entrance → Apartment → Owner 👥 Управление
на собственици и живущи

Системата поддържа информация за собствениците на апартаменти,
включително:

име на собственик телефонен номер свързан апартамент вход и блок, към
които принадлежи апартаментът

Това позволява по-лесна организация, бързо намиране на контактна
информация и по-добро администриране на входа.

💰 Финансов модул

Финансовият модул е една от основните части на системата. Чрез него се
управляват всички приходи, разходи, задължения и касови наличности.

Основни възможности:

каса за всеки вход записване на приходи от апартаменти записване на
разходи за входа автоматично изчисляване на задължения проследяване на
събрани и несъбрани такси поддръжка на различни категории разходи
обобщение на суми по входове

Пример за автоматично изчисляване на остатък:

RemainingSum = TotalSum - PaidSum

Колоната RemainingSum е computed column в SQL Server и не се въвежда
ръчно. Тя се изчислява автоматично от базата данни.

📊 Транзакции

Системата разделя транзакциите на два основни типа:

ApartmentTransactions

Използват се за записване на плащания и операции, свързани с конкретен
апартамент.

EntranceTransactions

Използват се за разходи и операции, свързани с целия вход.

Примери за категории:

Чистачка Асансьор Фонд ремонт Стълбище Събрани такси Несъбрана такса 🧾
Задължения

Системата позволява проследяване на задълженията на апартаментите чрез
таблицата Debts.

Всяко задължение съдържа:

апартамент обща сума платена сума оставаща сума

Оставащата сума се изчислява автоматично от SQL Server.

🛠 Поддръжка и ремонти

Модулът за поддръжка позволява записване на извършени ремонтни дейности
и разходи към конкретен вход.

Поддържат се:

описание на ремонта цена дата на ремонта вход, към който се отнася
ремонтът

Примери:

ремонт на асансьор боядисване ремонт на парапет подмяна на осветление 📢
Сигнали за проблеми

Живущите могат да подават сигнали за проблеми във входа.

Всеки сигнал съдържа:

заглавие описание потребител вход дата на създаване статус дали е
разрешен статус дали е редактиран

Това позволява по-добро проследяване на проблемите и тяхното решаване.

💬 Дискусии

Системата разполага с модул за дискусии, чрез който потребителите могат
да участват в комуникация, свързана с входа или сградата.

Възможности:

създаване на дискусии публикуване на отговори затваряне на дискусии
проследяване на дата на създаване връзка с потребител 📄 Документи

Модулът за документи позволява съхранение на информация за важни
файлове, свързани с входа.

Могат да се съхраняват:

договори фактури протоколи други документи

Всеки документ съдържа:

име на файл път до файла дата на качване вход, към който принадлежи ⚙️

Системата може да бъде заредена с примерни данни по два начина.

Вариант 1: Чрез SQL скрипт от SSMS

Ако имате генериран SQL скрипт от SQL Server Management Studio, който
съдържа CREATE TABLE и INSERT заявки, може да го използвате за създаване
и зареждане на базата.

Препоръчителни стъпки:

Отворете SQL Server Management Studio Свържете се към SQL Server
Създайте нова база данни или използвайте вече съществуваща Отворете New
Query Поставете SQL скрипта Проверете името на базата в USE
\[DatabaseName\] Стартирайте скрипта с Execute Вариант 2: Чрез готов
Create + Insert скрипт

Ако използвате скрипт, генериран от SSMS, е важно той да бъде подготвен
правилно за друга машина.

Примерно начало:

USE \[master\] GO

IF DB_ID(N'SyndiceoDB') IS NULL BEGIN CREATE DATABASE \[SyndiceoDB\] END
GO

USE \[SyndiceoDB\] GO

След това следват:

CREATE TABLE ... INSERT INTO ... CREATE INDEX ... ALTER TABLE ...
FOREIGN KEY ... Важни бележки при SQL скрипта

При таблици с IDENTITY колони трябва да се използва:

SET IDENTITY_INSERT \[dbo\].\[TableName\] ON

След INSERT заявките:

SET IDENTITY_INSERT \[dbo\].\[TableName\] OFF

Пример:

SET IDENTITY_INSERT \[dbo\].\[Addresses\] ON

INSERT \[dbo\].\[Addresses\] (\[AddressID\], \[Street\]) VALUES (1,
N'Марица')

SET IDENTITY_INSERT \[dbo\].\[Addresses\] OFF Computed колони

Computed колони не трябва да се попълват ръчно.

Например в таблицата Debts колоната RemainingSum се изчислява
автоматично:

\[RemainingSum\] AS (\[TotalSum\]-\[PaidSum\]) PERSISTED

Затова INSERT заявката трябва да съдържа само:

INSERT \[dbo\].\[Debts\] (\[Id\], \[ApartmentId\], \[TotalSum\],
\[PaidSum\]) VALUES (1, 1, CAST(60.00 AS Decimal(10, 2)), CAST(0.00 AS
Decimal(10, 2)))

Не трябва да се добавя RemainingSum в INSERT заявката.

Ред на зареждане на данните

При наличие на foreign key връзки е важно данните да се зареждат в
правилен ред.

Примерен ред:

1\. Addresses 2. Blocks 3. Entrances 4. Owners 5. Apartments 6.
Categories 7. Cashbox 8. Debts 9. Maintenance 10. Reports 11. Documents
12. Discussions 13. DiscussionReplies 14. Transactions

Ако foreign key ограниченията се създават след INSERT заявките, редът е
по-гъвкав, но все пак е препоръчително да се следва логическата
структура.

⚠️ Чести проблеми при стартиране на SQL скрипт Грешка при празна
стойност

Ако има INSERT от типа:

VALUES (1, 1, 1, N'', 0, 1, 2, 1, )

трябва да се поправи на:

VALUES (1, 1, 1, N'', 0, 1, 2, 1, NULL) Грешка при конкретен път до
MDF/LDF файл

Ако скриптът съдържа нещо подобно:

FILENAME = N'D:\Services\MSSQL\Data\db45189.mdf'

това може да не работи на друга машина.

По-добре е да се използва по-универсално създаване на база:

IF DB_ID(N'SyndiceoDB') IS NULL BEGIN CREATE DATABASE \[SyndiceoDB\] END
Грешка при Full-Text Catalog

Ако SQL Server няма инсталиран Full-Text Search, следният код може да
даде грешка:

CREATE FULLTEXT CATALOG

Ако проектът не използва реално full-text search, тази част може да бъде
премахната.
